# Dharmesh Shah - Sample Content Examples

This document contains examples of Dharmesh Shah's writing style, structure, and patterns based on his published content.

---

## Example 1: Ultra-Short LinkedIn Post

### Pattern: Contrarian Statement

**Example:**
> "Wanting work-life balance is OK. Wanting to build a breakthrough startup is OK. But you shouldn't expect both."

### Structure Analysis
- **Opening:** Two acceptable statements
- **Contrarian twist:** "But you shouldn't expect both"
- **Length:** Under 20 words
- **Impact:** Sparks discussion about trade-offs

### Key Elements
- Acknowledges both desires as valid
- Makes honest statement about reality
- No sugar-coating
- Standalone thought that works independently
- Provokes thought and discussion

## Example 2: Decision-Making Framework

### Pattern: Structured Framework

**Example:**
> "Here's my simple framework for judging ideas:
> 
> 1) Potential: If it worked, how big could it be?
> 2) Probability: What are the chances it'll work?
> 3) Proximity: How close is this to things I care about, know about, or am passionate about?"

### Structure Analysis
- **Opening:** Personal framework introduction
- **Framework:** Three clear criteria (3 P's)
- **Format:** Numbered list with questions
- **Length:** ~50 words

### Key Elements
- Personal ownership ("my simple framework")
- Clear structure (numbered list)
- Actionable questions for each point
- Easy to remember (3 P's)
- Practical application

## Example 4: Honest Reflection

### Pattern: Candid Admission

**Example:**
> "I plan to write here once every whenever. If I were a great writer, I would commit to some regular cadence of writing. But I'm not, so I won't."

### Structure Analysis
- **Opening:** Sets expectation (irregular schedule)
- **Self-deprecation:** Admits not being a great writer
- **Honesty:** Won't commit to something he can't deliver
- **Length:** ~30 words

### Key Elements
- Honest about limitations
- Self-deprecating humor
- Sets realistic expectations
- Relatable (many struggle with consistency)
- Builds trust through honesty

## Formatting Patterns

### Use of Formatting
- **Minimal formatting** - Plain text preferred
- **Numbered lists** for frameworks (sparingly)
- **No excessive emojis** - Use rarely if at all
- **Simple structure** - Let content speak
- **Line breaks** for readability (sparingly)

### Visual Structure
- Clean, uncluttered appearance
- Focus on message, not design
- Easy to read quickly
- Professional but casual
- No flashy elements
