## Sample 3: Medium Post (Framework - 400 words)

**Topic:** Document Intelligence Framework

**Post:**
> ## The Problem with Document Search
> 
> Most teams spend hours searching through documents. I've seen teams waste 10-15 minutes per query just trying to find information buried in PDFs, Word docs, and emails.
> 
> After researching document intelligence solutions and building my own RAG system, I found a better approach.
> 
> ## What Actually Works
> 
> Here's a 4-step framework for implementing document intelligence:
> 
> ### 1. Define Your Use Case
> - What specific questions do you need answered?
> - Which documents contain the answers?
> - Who needs access?
> 
> Start narrow. One use case. One document set. One user group.
> 
> ### 2. Choose Your Approach
> - **Simple:** Keyword search with better indexing
> - **Medium:** Semantic search with embeddings
> - **Advanced:** RAG with LLM integration
> 
> Don't over-engineer. Start with the simplest solution that works.
> 
> ### 3. Measure Impact
> - Time saved per query
> - Accuracy of results
> - User satisfaction
> 
> Track real metrics, not just technical performance.
> 
> ### 4. Iterate Based on Usage
> - What questions are users actually asking?
> - What documents are most accessed?
> - Where are the gaps?
> 
> Let real usage guide your improvements.
> 
> ## Key Takeaways
> 
> - Start with one use case, not everything
> - Use the simplest solution that works
> - Measure real impact, not complexity
> - Iterate based on actual usage
> 
> What document challenges are you facing?

**Analysis:**
- Problem-first approach
- Research-backed framework
- Clear structure with numbered steps
- Practical examples
- Actionable takeaways

## Sample 8: Framework Post (Medium - 350 words)

**Topic:** Evaluating AI Tools

**Post:**
> ## The AI Tool Evaluation Problem
> 
> There are hundreds of AI tools available. How do you know which ones are worth your time?
> 
> I've been testing AI tools for months, and I've developed a simple framework to evaluate them quickly.
> 
> ## The 3-Question Framework
> 
> Before investing time in any AI tool, ask:
> 
> ### 1. Does it solve a problem I actually have?
> Not a problem you might have. Not a problem someone else has. A problem you have right now.
> 
> If no, skip it.
> 
> ### 2. Is it significantly better than my current solution?
> "Better" means faster, cheaper, or higher quality. Not just "newer."
> 
> If it's only slightly better, it's probably not worth switching.
> 
> ### 3. Can I measure the impact?
> You need to know if it's working. That means measurable outcomes.
> 
> If you can't measure it, you can't improve it.
> 
> ## How to Use This
> 
> Run every AI tool through these three questions. If it doesn't pass all three, move on.
> 
> There are too many tools to try them all. Be selective.
> 
> ## Key Takeaway
> 
> Focus on tools that solve real problems, deliver measurable value, and fit your actual needs.
> 
> What tools have you found most valuable?

**Analysis:**
- Problem-first approach
- Structured framework
- Clear evaluation criteria
- Practical application
- Engagement question
